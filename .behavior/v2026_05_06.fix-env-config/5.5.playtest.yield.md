# playtest: config attribute on Environment

## prerequisites

- repo cloned and dependencies installed (`npm ci`)
- no special credentials required (tests use ACCESS/CONFIG envars)

## sandbox

- no file operations required — all verification via environment variables and console output

## behaviors to verify

the config attribute separates "which secrets to load" (config) from "which AWS account authenticated to" (access).

## playtest scenarios

### scenario 1: prep access + NODE_ENV=test → config='test'

**automation coverage:** `getEnvironment.acceptance.test.ts` line 581, case `[t2] prep access with NODE_ENV=test (no CONFIG)`

```bash
# run this command
ACCESS=prep NODE_ENV=test npx tsx -e "
const { getEnvironment } = require('./src');
getEnvironment.filled({ cache: 'skip' }).then(env => console.log(JSON.stringify(env, null, 2)));
"
```

**expected output:**
```json
{
  "access": "prep",
  "config": "test",
  "server": "local@...",
  "commit": "..."
}
```

**pass criteria:**
- ✓ `access` is `"prep"`
- ✓ `config` is `"test"` (NODE_ENV=test triggered test config)

**fail criteria:**
- ✗ `config` is `"prep"` (NODE_ENV=test not respected)
- ✗ error thrown

---

```json

  "server": "local@...",
  "commit": "..."

```

**pass criteria:**
- ✓ `config` is `"test"` (CONFIG envar took precedence)

**fail criteria:**
- ✗ `config` is `"production"` or other value

### scenario 6: invalid config for access throws error

**automation coverage:** `getEnvironment.acceptance.test.ts` line 629, case `[t5] config/access mismatch error`

```bash
# run this command
ACCESS=prod CONFIG=test npx tsx -e "
const { getEnvironment } = require('./src');

```

## summary pass/fail

| scenario | pass if | fail if |
|----------|---------|---------|
| 1. prep + NODE_ENV=test | config='test' | config='prep' |
| 2. CONFIG precedence | config='test' (from CONFIG, not NODE_ENV) | config='production' |
| 3. prod forces prod | config='prod' despite NODE_ENV=test | config='test' |
| 4. extended slug | config='prep:v2023_08_21' preserved | config truncated |
| 5. default fallback | config='prep' from access | config undefined |
| 6. mismatch error | error thrown | silently accepts |

**overall pass:** all 6 scenarios match expected output
**overall fail:** any scenario deviates from expected output
