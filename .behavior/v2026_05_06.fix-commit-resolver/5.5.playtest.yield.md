# playtest: sync + async commit resolver

## prerequisites

- node.js 18+ installed
- git repository (this repo)
- terminal with bash/zsh

## sandbox

all commands run in repo root. no file operations needed.

---

## happy paths

### path 1: sync commit detection via `getEnvironment.static()`

**action:**

```bash
npx tsx -e "
const { getEnvironment } = require('./src');
const env = getEnvironment.static({ cache: 'skip' });
console.log(JSON.stringify(env, null, 2));
"
```

**expected outcome:**

- output shows `{ access, commit, server }` object
- `commit` field is NOT null
- `commit` format is `$ref@$hash` (e.g., `v0.1.1@abc1234` or `vlad/fix-commit-resolver@abc1234`)

**pass criterion:** commit field populated without async/await

**test coverage:** `src/getEnvironment.acceptance.test.ts:527-545` (journey: static() with git commit detection)

---

### path 2: async commit detection via `getEnvironment()`

**action:**

```bash
npx tsx -e "
(async () => {
  const { getEnvironment } = require('./src');
  const env = await getEnvironment({ cache: 'skip' });
  console.log(JSON.stringify(env, null, 2));
})();
"
```

**expected outcome:**

- output shows `{ access, commit, server }` object
- `commit` field is NOT null
- `commit` format is `$ref@$hash`

**pass criterion:** commit field populated via async/await

**test coverage:** `src/domain.operations/parsers/factories.test.ts:7-22` (fromGit() returns async parser) + `src/domain.operations/parsers/commit/getEnvCommitFromGit.integration.test.ts` (git detection works)

---

### path 3: explicit `fromGit.sync()` parser

**action:**

```bash
npx tsx -e "
const { getEnvironment, fromGit, fromEnvar } = require('./src');
const env = getEnvironment.static({
  parsers: {
    commit: [fromGit.sync(), fromEnvar('COMMIT')],
  },
  cache: 'skip',
});
console.log('commit:', env.commit);
console.log('parser used: fromGit.sync()');
"
```

**expected outcome:**

- `commit` field is populated from git
- no COMMIT envar needed

**pass criterion:** `fromGit.sync()` returns parser that works in sync context

**test coverage:** `src/domain.operations/parsers/factories.test.ts:24-33`

---

### path 5: backwards compat — `fromGit()` remains async

**action:**

## edgey paths

### edge 1: COMMIT envar overrides git

**action:**

```bash
COMMIT=v1.0.0@override npx tsx -e "
const { getEnvironment } = require('./src');
const env = getEnvironment.static({ cache: 'skip' });
console.log('commit:', env.commit);
"
```

**expected outcome:**

- `commit` is `v1.0.0@override` (from envar, NOT from git)

**pass criterion:** envar takes precedence over git detection

**test coverage:** `src/getEnvironment.acceptance.test.ts:547-570`

---

### edge 2: dirty repo detection

**action:**

```bash
# check if repo has uncommitted changes
git status --porcelain | head -5
```

then:

```bash
npx tsx -e "
const { getEnvironment } = require('./src');
const env = getEnvironment.static({ cache: 'skip' });
console.log('commit:', env.commit);
console.log('has + suffix (dirty):', env.commit?.endsWith('+'));
"
```

**expected outcome:**

- if `git status --porcelain` shows changes, commit ends with `+`
- if no changes, commit has no `+` suffix

**pass criterion:** dirty state detected and appended to commit slug

**test coverage:** `src/domain.operations/parsers/commit/getEnvCommitFromGit.integration.test.ts`

## pass/fail criteria

### ✓ pass if

1. path 1: `getEnvironment.static()` returns commit without async/await
2. path 2: `getEnvironment()` returns commit with async/await
3. path 3: `fromGit.sync()` works in sync context
4. path 4: `fromGit.async()` works in async context
5. path 5: `fromGit()` returns async parser (backwards compat)
6. edge 1: COMMIT envar overrides git detection
7. edge 2: dirty repo shows `+` suffix in commit

### ✗ fail if

- any path returns `commit: null`
- `fromGit()` returns sync parser (breaks backwards compat)
- `fromGit.sync()` returns promise
- COMMIT envar is ignored when set
- dirty detection fails (no `+` suffix when changes exist)

all 5 happy paths and 2 edge paths have test coverage (acceptance, unit, or integration as appropriate). the playtest proves the experience matches the documented api.
