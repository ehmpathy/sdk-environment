# blackbox criteria: sync + async commit resolver

## usecase.1 = sync commit detection via getEnvironment.static()

```
given(user calls getEnvironment.static() in a git repository)
  when(defaults are used)
    then(commit is detected from git)
      sothat(sync users get commit traceability without explicit config)
    then(format is '$ref@$hash' or '$ref@$hash+' if dirty)
      sothat(commit slug is consistent with async behavior)

  when(repository is dirty)
    then(commit slug ends with '+')
      sothat(users know the build includes uncommitted changes)

  when(HEAD is tagged)
    then(ref is the tag name)
      sothat(release builds show version tags)

  when(HEAD is not tagged)
    then(ref is the branch name)
      sothat(dev builds show branch context)

given(user calls getEnvironment.static() outside a git repository)
  when(defaults are used)
    then(commit is null)
      sothat(non-git contexts fail gracefully)

  when(COMMIT envar is set)
    then(commit is the envar value)
      sothat(users can override via environment)

given(user calls getEnvironment.static() in a shallow clone)
  when(no tag history is available)
    then(ref falls back to branch name)
      sothat(CI/CD shallow clones work correctly)
    then(hash is still detected)
      sothat(commit traceability is preserved)
```

## usecase.3 = explicit parser selection

```
given(user wants sync commit detection)
  when(user passes fromGit.sync() to parsers.commit)
    then(commit is detected synchronously)
      sothat(users can explicitly opt into sync git exec)

given(user wants async commit detection)
  when(user passes fromGit() to parsers.commit)
    then(commit is detected asynchronously)
      sothat(users can parallelize git exec with other init)

  when(user passes fromGit.async() to parsers.commit)
    then(commit is detected asynchronously)
      sothat(explicit async variant is available for symmetry)
```

## usecase.4 = cache behavior

```
given(user calls getEnvironment.static() multiple times)
  when(cache is not skipped)
    then(git exec happens only once)
      sothat(the 50-200ms cost is one-time per process)
    then(same object is returned)
      sothat(identity comparison works)

given(user calls getEnvironment.filled() multiple times)
  when(cache is not skipped)
    then(git exec happens only once)
    then(same object is returned)

given(user needs fresh parse)
  when(user passes { cache: 'skip' })
    then(git exec happens again)
      sothat(users can force re-detection if needed)
```

## usecase.5 = backwards compatibility

```
given(extant code uses fromGit())
  when(user upgrades sdk-environment)
    then(fromGit() still returns async parser)
      sothat(extant code continues to work)
    then(no migration is required)
      sothat(upgrade is non-disruptive)
```
