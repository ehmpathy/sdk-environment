## decomposition analysis

**dimension count**: 4 independent variables in matrix 1 (method, context, HEAD state, dirty)

**is this too broad?** no — the dimensions are tightly coupled:
- context determines whether git commands work
- HEAD state and dirty are orthogonal git properties
- method is the API surface

all 4 dimensions are necessary to specify commit detection behavior. no decomposition needed.

**observation**: static() and filled() have identical outcomes for commit detection. the only difference is sync vs async execution, which is an implementation detail not visible in blackbox outcomes.
