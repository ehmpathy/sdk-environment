# blocker.1: Global cache ignores input parsers, causing unpredictable behavior

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.behavior-hazards.md

**locations**:
- src/domain.operations/getEnvironment/getEnvironment.ts:184
- src/domain.operations/getEnvironment/getEnvironment.ts:265

The caching mechanism uses a fixed cache key ('filled' or 'static') regardless of the input parsers, leading to non-deterministic behavior. Once the cache is populated with a result from one set of parsers, subsequent calls with different parsers will return the cached result instead of recomputing based on the new inputs. This violates the rule against non-determinism and unexpected behavior, as the function should produce consistent outputs for the same inputs but here it depends on call order and prior cache state.
To fix, the cache key must incorporate the input configuration (e.g., serialize based on a hash of the parsers) or disable caching when custom parsers are provided.

**snippet**:
```ts
const computeFilledEnvironmentCached = withSimpleCaching(
  computeFilledEnvironment,
  {
    cache: filledCache,
    serialize: { key: () => 'filled' },
  },
);
```

# blocker.2: Cache population with 'skip' may cause race conditions

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.behavior-hazards.md

**locations**:
- src/domain.operations/getEnvironment/getEnvironment.ts:200

When cache is set to 'skip', the function computes the result and manually sets the cache with filledCache.set('filled', promise). In concurrent environments, multiple calls with 'skip' could lead to race conditions where the cache is overwritten or inconsistent. This creates unpredictable behavior and potential data corruption, violating the rule against race conditions and hidden side effects.
To fix, ensure atomic cache operations or synchronize cache access.

**snippet**:
```ts
  // skip cache read but still populate cache for subsequent calls
  if (input?.cache === 'skip') {
    const promise = computeFilledEnvironment(config);
    filledCache.set('filled', promise);
    return promise;
  }
```
