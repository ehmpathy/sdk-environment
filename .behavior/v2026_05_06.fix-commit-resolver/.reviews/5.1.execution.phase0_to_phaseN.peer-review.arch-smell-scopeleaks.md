# blocker.1: Decode-friction in orchestrator functions

**rule**: .agent/repo=ehmpathy/role=architect/briefs/practices/rule.forbid.decode-friction-in-orchestrators.md

**locations**:
- src/domain.operations/getEnvironment/getEnvironment.ts

The 'filled' and 'static' functions in getEnvironment.ts act as orchestrators but contain inline logic and helper functions that require mental simulation to understand. For example, 'asParsersNamed' and 'getOneFromParsersAsync' are defined inline, forcing readers to decode their implementation rather than grasping intent from named calls. According to the rule, orchestrators must read as narrative without decode-friction; this logic should be extracted to named transformers or communicators. This violates the separation where orchestrators compose leaf operations, not contain them.

**snippet**:
```ts
const filled = (
  input?: {
    parsers?: {
      access?: AsyncParser<EnvironmentAccessTier>[] | null;
      server?: AsyncParser<EnvironmentServerTier>[] | null;
      commit?: AsyncParser<EnvironmentCommitSlug>[] | null;
    } | null;
    cache?: 'skip' | null;
  } | null,
): Promise<Environment> => {
  const config = {
    accessParsers: input?.parsers?.access ?? DEFAULT_ACCESS_PARSERS,
    serverParsers: input?.parsers?.server ?? DEFAULT_SERVER_PARSERS,
    commitParsers: input?.parsers?.commit ?? DEFAULT_COMMIT_PARSERS,
  };

  // skip cache read but still populate cache for subsequent calls
  if (input?.cache === 'skip') {
    const promise = computeFilledEnvironment(config);
    filledCache.set('filled', promise);
    return promise;
  }

  return computeFilledEnvironmentCached(config);
};
```

# blocker.2: Inline helpers causing decode-friction in orchestrators

**rule**: .agent/repo=ehmpathy/role=architect/briefs/practices/rule.forbid.decode-friction-in-orchestrators.md

**locations**:
- src/domain.operations/getEnvironment/getEnvironment.ts

Helpers like 'asParsersNamed' and 'getOneFromParsersAsync' are defined within the orchestrator file, requiring readers to scan the file to understand what they do (e.g., 'asParsersNamed' maps parsers to named objects, 'getOneFromParsersAsync' runs a parser chain until success). This breaks the narrative flow of orchestrators, which should delegate to named operations. Extract these to separate transformer files to make the orchestrator readable at the call site.

**snippet**:
```ts
const asParsersNamed = <T>(
  parsers: Array<() => T>,
): Array<{ parser: () => T; name: string }> =>
  parsers.map((parser, index) => ({
    parser,
    name: computeParserName({ parser, index }),
  }));

const getOneFromParsersAsync = async <T>(
  parsers: Array<{ parser: AsyncParser<T>; name: string }>,
  validate: (value: unknown) => value is T,
  attributeName: string,
): Promise<T> => {
  for (const { parser, name } of parsers) {
    const result = await parser();
    if (result !== null) {
      if (!validate(result)) {
        throw new BadRequestError(
          `invalid ${attributeName} value: ${JSON.stringify(result)}. parser: ${name}`,
        );
      }
      return result;
    }
  }
  const parserNames = parsers.map((p) => p.name);
  throw new UnexpectedCodePathError(
    `could not derive ${attributeName}. tried parsers: ${parserNames.join(', ')}`,
  );
};
```

# nitpick.1: Operation name does not strictly follow get-set-gen pattern

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/evolvable.domain.operations/rule.require.get-set-gen-verbs.md.min

**locations**:
- src/domain.operations/getEnvironment/getEnvironment.ts

The main operation 'getEnvironment' uses 'get', but the rule specifies 'getOne*' or 'getAll*' for retrieval operations to indicate cardinality. While 'getEnvironment' is a valid retrieval, it could be renamed to 'getOneEnvironment' for consistency with the pattern. This is a nitpick as it may be intentional for a singleton-like parser, but aligning with the rule improves uniformity.

**snippet**:
```ts
/**
 * .what = get environment - callable directly or via .filled()/.static()
 * .why = main entry point for sdk-environment
 */
interface GetEnvironment {
  (input?: Parameters<typeof filled>[0]): Promise<Environment>;
  filled: typeof filled;
  static: typeof staticEnv;
}

const getEnvironmentFn: GetEnvironment = Object.assign(
  // direct call delegates to filled
  (input?: Parameters<typeof filled>[0]): Promise<Environment> => filled(input),
  {
    /**
     * async, runs all parsers (aws and git), cached
     */
    filled,

    /**
     * sync, skips async parsers, cached independently
     */
    static: staticEnv,
  },
);

export const getEnvironment = getEnvironmentFn;
```
