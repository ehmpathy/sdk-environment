# feat: add account name fallback for env access detection

## context

`getEnvAccessFromAwsAccountAlias` resolves environment tier (prod/prep/test) from AWS account alias via `iam:ListAccountAliasesCommand`. however, many accounts don't have an alias set.

## proposal

add `getEnvAccessFromAwsAccountName` as a fallback parser via the Account service API.

## considerations

### pros
- name is always present (required at account creation)
- simple permission: `account:GetAccountInformation` (read-only, own account)
- no Organizations membership required
- dynamic import keeps bundle small

### cons
- account names aren't globally unique
- name might match pattern by accident (e.g., someone named their account "my-prod-experiments")
- less intentional than alias (alias is typically set specifically for this purpose)

### open question

is accidental name match a real risk? within your own AWS org you control name conventions, so probably fine. alias-first order mitigates this ŌĆö alias takes precedence when set.
