# roadmap: cli binary for commit

## overview

five phases to implement the cli binary:

1. package.json infrastructure
2. cli contract implementation
3. binary infrastructure
4. integration tests
5. acceptance tests

---

## phase 0: package.json infrastructure

### briefread

none required — changes are declarative.

### deliverables

- [ ] update package.json with bin entry
- [ ] update package.json with files array (include /bin)
- [ ] add build:compile:bun command

### acceptance criteria

```
given package.json is updated
  when user runs `npm pack --mode plan`
    then output includes bin/run
    then output includes bin/run.bun.sdk-environment.bc
```

### verification

```bash
npm pack | grep -E 'bin/run'
```

## phase 1: cli contract implementation

### briefread

- `.behavior/v2026_05_08.fix-commit-parser/3.3.1.blueprint.product.yield.md` (implementation notes)
- `.behavior/v2026_05_08.fix-commit-parser/1.vision.yield.md` (exit codes)

### deliverables

- [ ] create src/contract/cli/invoke.ts (CLI router)
- [ ] create src/contract/cli/getCommit.ts (get commit command)

### acceptance criteria

```
given invoke.ts and getCommit.ts exist
  when invoke.ts is executed with `get commit` args
    then getCommit() is called
  when invoke.ts is executed with unknown args
    then exit code is 2
    then stderr contains "unknown command"
```

```
given getCommit.ts exists
  when executed in a git repo with a tag at HEAD
    then stdout contains `$tag@$hash`
    then exit code is 0
  when executed in a git repo on a branch (no tag)
    then stdout contains `$branch@$hash`
    then exit code is 0
  when executed outside a git repo
    then stderr contains error message
    then exit code is 2
```

### verification

```bash
npx tsx ./src/contract/cli/invoke.ts get commit
```

### briefread

- `.behavior/v2026_05_08.fix-commit-parser/3.3.1.blueprint.product.yield.md` (bash shim pattern)
- ehmpathy/rhachet bin/run (reference pattern)

- [ ] create bin/run (bash shim)

### acceptance criteria

```
given bin/run exists with correct symlink resolution
  when user runs `./bin/run get commit`
    then stdout contains commit slug
    then exit code is 0
```

```
given binary is symlinked from node_modules/.bin/
  when user runs `npx sdk-environment get commit`
    then stdout contains commit slug
    then exit code is 0
```

### verification

```bash
npm run build:compile:bun
./bin/run get commit
```

## phase 4: acceptance tests

### briefread

- `.behavior/v2026_05_08.fix-commit-parser/2.1.criteria.blackbox.yield.md` (all episodes)
- `.behavior/v2026_05_08.fix-commit-parser/2.2.criteria.blackbox.matrix.yield.md` (matrix cases)

### deliverables

- [ ] create blackbox/cli/get-commit.acceptance.test.ts (isolated cases with snapshots)
- [ ] create blackbox/cli/get-commit.journey.test.ts (developer release workflow)

### acceptance criteria

```
given acceptance tests exist
  when tests are executed
    then all matrix cases pass:
      - tag + clean → v1.0.0@abc1234
      - tag + dirty → v1.0.0@abc1234+
      - branch + clean → main@abc1234
      - branch + dirty → main@abc1234+
      - detached + clean → abc1234@abc1234
      - detached + dirty → abc1234@abc1234+
      - not a git repo → exit 2
      - unknown command → exit 2
    then all outputs are snapshotted
```

```
given journey test exists
  when developer release workflow is simulated
    then t0: feature branch returns branch slug
    then t1: after tag, returns tag slug (precedence)
    then t2: after dirty change, returns dirty suffix
    then t3: outside repo, returns constraint error
```

### verification

```bash
rhx git.repo.test --what acceptance --scope get-commit --mode apply
```
