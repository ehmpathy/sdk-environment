# blackbox criteria: cli binary for commit

## usecase.1 = get commit slug at deploy time

### episode: basic usage

```
given a git repo with commits
  when user runs `npx sdk-environment get commit`
    then stdout contains a commit slug in format `$gitref@$hash`
      sothat they can capture it to an env var
    then exit code is 0
      sothat downstream commands proceed
```

### episode: tag-based deploy

```
given a git repo checked out at a tag (e.g., v1.2.3)
  when user runs `npx sdk-environment get commit`
    then stdout contains the tag as gitref (e.g., `v1.2.3@abc1234`)
      sothat deployed code is identifiable by version
```

### episode: branch-based deploy

```
given a git repo checked out on a branch (e.g., main)
  given no tag at HEAD
  when user runs `npx sdk-environment get commit`
    then stdout contains the branch as gitref (e.g., `main@abc1234`)
      sothat deployed code is identifiable by branch
```

### episode: dirty worktree

```
given a git repo with uncommitted changes
  when user runs `npx sdk-environment get commit`
    then stdout contains the dirty suffix (e.g., `main@abc1234+`)
      sothat developers know the deploy includes uncommitted work
```

## usecase.2 = handle error conditions gracefully

### episode: not a git repo

```
given a directory that is not a git repo
  when user runs `npx sdk-environment get commit`
    then stderr contains a clear error message
    then exit code is 2 (constraint)
      sothat caller knows to fix the environment
```

### episode: git command fails

```
given git is not available or fails unexpectedly
  when user runs `npx sdk-environment get commit`
    then stderr contains the error details
    then exit code is 1 (malfunction)
      sothat caller knows to investigate
```

## usecase.3 = edge cases

### episode: detached HEAD

```
given a git repo in detached HEAD state
  given no tag at HEAD
  when user runs `npx sdk-environment get commit`
    then stdout contains the commit hash as gitref
      sothat deployed code is still identifiable
```

### episode: tag precedence over branch

```
given a git repo on branch main
  given a tag v1.0.0 at HEAD
  when user runs `npx sdk-environment get commit`
    then stdout contains the tag as gitref (e.g., `v1.0.0@abc1234`)
      sothat semantic versions take precedence in deploy identity
```
