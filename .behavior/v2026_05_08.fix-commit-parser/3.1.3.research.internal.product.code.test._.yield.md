## pattern.1 = integration tests for getEnvCommitFromGit

**location**: `src/domain.operations/parsers/commit/getEnvCommitFromGit.integration.test.ts`

**[REUSE]** — follow this pattern for CLI integration tests

### relevant code

```typescript
// line 1-25: integration test with given/when/then
import { given, then, when } from 'test-fns';

describe('getEnvCommitFromGit', () => {
  given('a git repository', () => {
    when('commit info is requested', () => {
      then('returns commit slug in $ref@$hash format', async () => {
        const result = await getEnvCommitFromGit();

        // verify not null
        expect(result).not.toBe(null);

        // verify contains @ separator
        expect(result).toContain('@');

        // verify matches expected format: ref@hash or ref@hash+ (dirty)
        const pattern = /^.+@[a-z0-9]+\+?$/i;
        expect(pattern.test(result!)).toBe(true);

        // snapshot format structure (ref@hash pattern, not exact value)
        expect(result?.replace(/@[a-z0-9]+\+?$/i, '@<hash>')).toMatchSnapshot();
      });
    });
  });
```

### key patterns

1. **BDD structure**: `given/when/then` from test-fns
2. **format verification**: regex pattern `/^.+@[a-z0-9]+\+?$/i`
3. **snapshot with hash normalization**: replace hash with `<hash>` for stable snapshots
4. **sync/async parity**: separate test blocks verify both variants return same result
