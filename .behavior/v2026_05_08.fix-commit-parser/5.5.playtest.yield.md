# playtest: sdk-environment get commit CLI

## prerequisites

- node 18+ installed
- this repo checked out
- dependencies installed (`npm ci`)

## sandbox

all playtest steps operate on the current repo state. no file creation needed — the CLI reads from git.

## happy paths

### 1. basic invocation in this repo

```bash
npx tsx ./src/contract/cli/invoke.ts get commit
```

**expected outcome:**
- stdout: single line in format `$ref@$hash` or `$ref@$hash+`
- pattern: `v{semver}@{7-char-hash}` or `{branch}@{7-char-hash}`
- if worktree is dirty: ends with `+`
- exit code: 0

**acceptance test:** `blackbox/cli/get-commit.acceptance.test.ts` cases 1-4

### 2. verify dirty suffix

first, check if worktree is clean or dirty:
```bash
git status --porcelain
```

if output is non-empty (dirty), the CLI output should end with `+`.

```bash
npx tsx ./src/contract/cli/invoke.ts get commit | grep -E '\+$'
```

**expected outcome:**
- if dirty: matches (exit 0)
- if clean: no match (exit 1)

**acceptance test:** `blackbox/cli/get-commit.acceptance.test.ts` cases 2, 4

### 3. capture to env var

```bash
COMMIT=$(npx tsx ./src/contract/cli/invoke.ts get commit)
echo "captured: $COMMIT"
```

**expected outcome:**
- variable contains the commit slug
- no extra whitespace or newlines
- usable for deploy scripts

**acceptance test:** this tests the core use case from vision

## edgey paths

### 4. unknown command

```bash
npx tsx ./src/contract/cli/invoke.ts foo bar
echo "exit code: $?"
```

**expected outcome:**
- stderr: `unknown command`
- stdout: empty
- exit code: 2 (constraint)

**acceptance test:** `blackbox/cli/get-commit.acceptance.test.ts` case 6

### 5. not a git repo (simulated)

```bash
mkdir -p .temp/not-a-repo
cd .temp/not-a-repo
npx tsx ../../src/contract/cli/invoke.ts get commit
echo "exit code: $?"
cd ../..
rm -rf .temp/not-a-repo
```

**expected outcome:**
- stderr: contains "not a git repo" or "no ref found"
- stdout: empty
- exit code: 2 (constraint)

**acceptance test:** `blackbox/cli/get-commit.acceptance.test.ts` case 5

## pass/fail criteria

✓ **pass if:**
- step 1 outputs a valid commit slug in `$ref@$hash+?` format
- step 2 dirty suffix matches worktree state
- step 3 variable capture works without extra whitespace
- step 4 unknown command returns exit 2
- step 5 non-repo returns exit 2

✗ **fail if:**
- any step returns unexpected exit code
- output format deviates from `$ref@$hash+?`
- dirty suffix does not reflect worktree state
- error messages go to stdout instead of stderr
