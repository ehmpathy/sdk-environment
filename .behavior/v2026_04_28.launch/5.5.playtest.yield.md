## playtest: sdk-environment

### prerequisites

- node.js 18+ installed
- git repository with at least one commit
- terminal access
- run all commands from the sdk-environment repo root (where package.json lives)

### sandbox

all file operations target `@gitroot/.temp/` — no files persist outside this folder.

---

### happy paths

#### 1. basic usage with explicit env vars

**action:**
```sh
# run from sdk-environment repo root
ACCESS=prod SERVER=cloud@aws.lambda COMMIT=v1.2.3@abc123 npx tsx -e "
import { getEnvironment } from './src/index';

const env = await getEnvironment({ cache: 'skip' });
console.log(JSON.stringify(env, null, 2));
"
```

**expected outcome:**
```json
{
  "access": "prod",
  "server": "cloud@aws.lambda",
  "commit": "v1.2.3@abc123"
}
```

**acceptance test:** `src/getEnvironment.acceptance.test.ts` — `getEnvironment() is called directly → returns Environment (README contract)`

### edgey paths

#### 1. all parsers return null

**action:**
```sh
# run from sdk-environment repo root
npx tsx -e "
import { getEnvironment } from './src/index';

try {
  await getEnvironment.filled({
    cache: 'skip',
    parsers: {
      access: [() => null],
      server: [() => null],
      commit: [() => null],
    },
  });
} catch (error) {
  console.log('error:', error.message);
}
"
```

**expected outcome:**
- error message contains `could not derive access`
- process exits cleanly

**acceptance test:** `src/getEnvironment.acceptance.test.ts` — `filled() is called with empty parsers → throws error with snapshot message`

#### 4. dirty commit indicator

**action:**
```sh
# run from sdk-environment repo root
npx tsx -e "

const env = await getEnvironment({
  cache: 'skip',
  parsers: {
    access: [() => 'test'],
    server: [() => 'local@unix'],
    commit: [fromGit()],  // real git parser
  },
});
console.log('commit:', env.commit);
// if uncommitted changes exist: ends with +
// otherwise: clean hash
"
```

**expected outcome:**
- commit format is `$ref@$hash` or `$ref@$hash+`
- the `+` suffix indicates uncommitted changes

**acceptance test:** `src/domain.operations/parsers/commit/getEnvCommitFromGit.integration.test.ts` — real git detection

### pass/fail criteria

- **pass if:**
  - all happy path commands produce expected JSON output
  - all edge case commands produce expected error messages
  - no command hangs or crashes unexpectedly
  - output matches the documented README contract

- **fail if:**
  - any command produces unexpected output
  - any error message is unclear or absent
  - any command hangs indefinitely
  - environment values do not match explicit env vars
