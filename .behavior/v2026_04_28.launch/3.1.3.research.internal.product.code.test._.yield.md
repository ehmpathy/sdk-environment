## test classification for sdk-environment

### unit tests (*.test.ts)

for pure parsers and transformers:
- `fromEnvar()` with mocked `process.env`
- `fromNodeEnv()` with mocked `NODE_ENV`
- validation logic for access/server values
- type guards like `isAccessTier()`

### integration tests (*.integration.test.ts)

for parsers that touch external systems:
- `fromGit()` — needs real git repo
- `fromAwsAccountAlias()` — needs aws credentials
- `fromLambdaTaskRoot()` — needs lambda-like env

### acceptance tests (*.acceptance.test.ts)

for blackbox contract:
- `getEnvironment.filled()` returns valid Environment
- `getEnvironment.static()` returns valid Environment
- cache behavior works as specified
- error messages are clear
