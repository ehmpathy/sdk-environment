### [REUSE] domain-glossaries — typed glossary pattern

```ts

export type AsOfGlossary<T, G extends string, R = true> =
```

[3] domain-glossaries/src/AsOfGlossary.ts

- pit-of-success: invalid values fail at type level

### [EXTEND] parser chain pattern

readme shows first-wins parser chain:

```ts
// readme.md:109-127
const environment = await getEnvironment({
  access: [
    getEnvAccess.fromEnvar,           // explicit override
    getEnvAccess.fromAwsAccountAlias, // infer from aws
    getEnvAccess.fromNodeEnv,         // fallback
  ],
  ...
});
```
[5] readme.md

**how this relates:**
- parsers are `() => T | null | Promise<T | null>`
- first non-null wins
- needs both sync and async parser support

### [EXTEND] static vs filled pattern

vision decided on:
- `getEnvironment.static()` — skips async parsers
- `getEnvironment.filled()` — runs all parsers

this pattern doesn't exist in other ehmpathy packages — new for sdk-environment.
